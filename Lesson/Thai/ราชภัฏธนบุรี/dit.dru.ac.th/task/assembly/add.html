บทเรียน ADD(ADDITION)

 

     คำอธิบาย

การปฎิบัติการ

     : เป็นการบวก OPERAND 2 ตัว

:คำสั่ง ADD เป็นการบวก OPERAND 2 ตัว โดยผลลัพธ์เก็บที่ OPERANDตัวสุดท้าย

SYNTAX : ADD OPERAND ตัวสุดท้าย OPERAND ตัวแรก

ตัวอย่าง :

ADD AX,BX           บวก REGISTER กับ REGISTER
ADD EBX,ECX           บวก REGISTER กับ REGISTER
ADD EXTMEN,AX           บวก REGISTER กับ หน่วยความจำ
ADD OX,BUFF           บวก หน่วยความจำ กับ REGISTER
ADD AL,4           OPREATE แบบ IMMEDIATE ไปยัง ACCUMULATOR
ADD EAX,98765432H           (ใช้ใน 80386) OPREATE แบบ IMMEDIATE ไปยังACCUMULATOR
ADD CX,1985           OPREATE แบบ IMMEDIATE ไปยัง REGISTER
ADD EXTMEM,23           OPREATE แบบ IMMEDIATE หน่วยความจำ

ตัวอย่าง : Program Add;

While_P segment

assume cs:while_p,ds:while_p ; assume code segment

แฟลกที่ถูกกระทำ : AF,CF,OF,PF,SF.ZF

Auxiliary Carry Flag (AF) ใช้สำหรับการคำนวนเลขที่มีการใช้ตัวทด หรือ ตัวยืม สำหรับเลข 1 หลักขนาด 4 Bit ของ BCD

Carry Flag (CF) มีค่าเป็น 1 เมื่อมีตัวทดหรือตัวยืม ในขณะทำการคำนวณทางคณิตศาสตร์ ด้วยข้อมูล8 หรือ 16 Bit ถ้ามีคำสั่ง Shift,Rorate เกิน Register ก็จะให้มี CF นี้ด้วยมีค่าเป็น 0 ในกรณีอื่นนอกจากนี้

Parity Flag (PF) ใช้สำหรับการติดต่อข้อมูล

มีค่าเป็น 1 ได้ Parity เป็นเลขคี่

มีค่าเป็น 0 ได้ Parity เป็นเลขคู่

OverFlow (OF) มีค่าเป็น 1 เมื่อผลลัพธ์ที่ได้มีค่าเกินBit สูงของข้อมูล

มีค่าเป็น 0 เมื่อผลลัพธ์ที่ได้มีค่าไม่เกินBit สูงของข้อมูล

Parity Flag (PF) ใช้สำหรับในการติดต่อข้อมูล

Sign Flag (SF) มีค่าเป็น 1 เมื่อผลลัพธ์เป็นค่าลบ

มีค่าเป็น 0 เมื่อผลลัพธ์เป็นค่าบวก

Zero Flag (ZF) มีค่าเป็น 1 เมื่อผลลัพธ์ที่ได้มีค่าเป็น 0

มีค่าเป็น 0 ในกรณีอื่น นอกจากนี้

แฟลกที่ไม่ต้องกำหนด : ไม่มี

ข้อยกเว้นใน MODE PROTECTED : จะยกเว้นในกรณีที่อยู่ใน SEGMENT

ที่เขียนข้อมูลไม่ได้ถ้าเป็นกรณีที่ใช้ SEGMENT CS,DS,BS จะใช้คำสั่งนี้ใน MODE นี้ได้ แต่ถ้าใช้ SEGMENT SS จะเกิดความผิดพลาดขึ้น

ข้อยกเว้นใน MODE REAL ADDRESS : จะเกิด INT 13 ขึ้นเมื่อ OPERANDอยู่ในหน่วยความจำOFFFFH

 

คำสั่ง Add Whit Carry (ADC)

คำสั่ง ADC ทำงานเช่นด้วยกับคำสั่ง ADD แต่จะรวม carry flag (CF) ด้วยคือ

destination = destination + source + carry

ผู้ใช้อย่างไม่เข้าใจเกี่ยวกับการใช้ carry flag ในเครื่องคอมพิวเตอร์ แต่จริงได้มีการใช้ carry ในการบวกเลขฐานสิบมานานแล้ว เข่น การบวก

36

79

42

157

ผู้ใช้จะบวกเลขที่อยู่ในหลักด้วยกัน เช่น ตำแหน่งทางขวามือสุดคือหลักหน่วย เอา 6+9+2 ได้ 17 ใส่ 7 ในหลักหน่วยและทดค่า1 ในตำแหน่งถัดไป คือ หลักสิบ ค่า 1 ที่ทดไป คือ carry นั่นเอง ในตำแหน่งถัดไป คือ หลักสิบ เอา 3+7+4+1 (carry) ได้ 15 ใส่ค่า 5 ในหลักสิบ และทดค่า 1 ไปใส่ไว้ในหลักร้อย จะเห็นว่าเกิด carry ในการบวก เมื่อผลลัพธ์ที่ได้มากกว่าค่าของเลขโดดสูงสุดที่ตำแหน่งนั้นเขียนได้

ตัวอย่างของการใช้คอมพิวเตอร์บวกเลขฐานสอง 2 ค่า คือ 250 และ 10 จะได้

+

0000 1010

ผลลัพธ์ที่ได้คือค่า 260 ถ้าทำการบวกโดยใช้วิธีรีจิสเตอร์ชนิด 8 บิต จะได้ผลลัพธ์ได้บิตที่ 0-7ใน 8 บิต รีจิสเตอร์ และบิตที่ 8อยู่ใน carry flag ดังนั้น ในการบวกเลขจำนวนมากๆ สามารถทำได้โดยใช้คำสั่ง ADD คู่กับ ADC เช่นการบวกเลขขนสด 32 บิต จะบวก 16 บิตล่างโดย ใช้คำสั่ง ADD และบวก 16 บิตบนโดยใช้คำสั่ง ADC

ADD AC,CX ; Add low order 16 bits

ADC BX,DX ; then high order 16 bits

ชนิดของคำสั่ง OPERAND ของคำสั่ง ADD,ADC นอกจากจะเป็นรีจิสเตอร์ 2 อันแล้วยังมีรีจิสเตอร์และค่าในหน่วยความจำ หรือค่าในหน่วยความจำและรีจิสเตอร์ หรือรีจิสเตอร์และค่าตัวเลข หรือค่าในหน่วยความจำและค่าตัวเลข ดังตัวอย่าง

ADD M_WORD,AX

ADD AL,25

ADD M_BYTE,0FH

คำสั่ง ASCII และ Decimal Adjust For Addition (AAA และ ADD)

โปรเซสเซอร์ 8088 จะบวกเลขเหมือนกับว่าเป็นเลขฐานสองหมด ดังนั้น ถ้าค่าที่นำมาใช้เป็นเลขฐานสิบจะเกิดอะไรขึ้น มาสำรวจโดยใช้ตัวอย่างการบวกค่าฐานสิบ คือ 25 กับ 56 จะได้ว่า

0010 0101 (BCD 25)

+

0101 0110 (DCD 56)

0111 1011 (=??)

จะเห็นได้ว่าผลลัพธ์ที่ได้แทนจะเป็นค่า BCD 81 กลับค่าได้ 7B H แต่ผลลัพธ์ได้กับแตกต่างกันนั้นไม่ได้หมายความว่าไม่สามารถบวกเลข BCD กับ 8088 ได้ แต่หมายความว่าต้องการมีการปรับค่าของผลลัพธ์ให้อยู่ในเลขฐานสิบ หรือ BCD ให้ได้

คำสั่ง AAA และ DAA จะปรับค่าของผลลัพธ์จากการบวกเลขฐานสิบโดยที่ผลลัพธ์ที่จะปรับค่าต้องอยู่ในรีจิสเตอร์ AL

คำสั่ง AAA จะปรับค่าในรีจิสเตอร์ AL เป็น unpacked degit โดยการพิจารณาค่า 4 บิตล่างของรีจิสเตอร์ AL ถ้ามีค่าระหว่าง 0-9 ก็จะทำให้ 4 บิตบนของรีจิเตอร์ AL มีค่า 0 แฟลก AF และ CF ก็จะมีค่า 0 ด้วยแต่ถ้า 4 บิตล่างของรีจิสเตอร์ AL มีค่ามากกว่า 9 หรือแฟลก AF มีค่า 1 ก็จะมีวิธีการปรับค่าดังนี้

- บวก 6 กับ รีจิสเตอร์ AL

- บวก 1 กับ รีจิสเตอร์ AH

- แฟลก AF และ CF มีค่า 1

- 4 บิตบนของรีจิสเตอร์ AL มีค่า 0

คำสั่ง DAA จะเปลี่ยนค่าในรีจิสเตอร์ AL เป็น PACKED DECI?AL DIGIT 2 ค่าคำสั่ง DAA มีการทำงานคล้ายคำสั่ง AAA เพียงแต่ต้องพิจารณาตัวเลข 2 ตัว วิธีการปรับค่าของคำสั่ง DAA มีดังนี้

- ถ้า 4 บิต ล่างของรีจิสเตอร์ AL มีค่ามากกว่า 9 หรือ แฟลก AF มีค่า 1 บวกค่า 6 กับ รีจิสเตอร์ AL และให้ค่าแฟลก AF เป็น 1

- ถ้า 4 บิต บนของรีจิสเตอร์ AL มีค่ามากกว่า 9 หรือ แฟลก AF มีค่า 1 บวกค่า 60H กับ รีจิสเตอร์ AL และให้ค่าแฟลก AF เป็น 1

คำสั่ง DAA มีที่ใช้ดังนี้

ADD AL,BL ;add packed BCD numbers in AL and BL

DAA ;and make the result a packed number
