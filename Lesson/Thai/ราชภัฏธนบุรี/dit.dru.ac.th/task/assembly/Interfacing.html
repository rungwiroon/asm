      ระหว่างภาษา

INTERFACING SUBROUTINES

เมื่อต้องการติดต่อ SUBROUTINES ภาษา ASSEMBLY กับ ภาษาระดับสูง ต้องรู้สิ่งนี้ คือ

1. โปรแกรมย่อย (SUBROUTINES) ที่ถูกเรียกใช้เป็นอย่างไร
2. พารามิเตอร์ (PARAMETERS) ที่ถูกส่งผ่านค่าเป็นอย่างไร
3. ค่า (VALUES) ที่ถูกส่งกลับเป็นอย่างไร

 

PASSING PARAMETERS

โดยทั่วๆไป ใช้ STACK ในการผ่านค่าพารามิเตอร์ไปยังโปรแกรมย่อย

        
พารามิเตอร์ที่ถูกส่งไป STACK เป็นอย่างไร        

1. ปกติพารามิเตอร์ จะส่งเป็นตัวเลขแบบ SHORT INTEGER

        

INTEGER = 16 BITS หรือ สั้นกว่ากรณีเป็น SHORT INTEGER (INTEGER กับ SHORT INTEGER

เวลาเก็บจริงๆ จะ 2 BYTES เท่านั้นแต่ SHORT INTEGER จะเร็วกว่า INTEGER กรณีในเครื่อง MICROCOMPUTER)

        

2. พารามิเตอร์ที่ส่งเป็นตัวอักษร

       

STRING = POINTER ที่ STRING ถูกส่งผ่าน

        
ACCESSING PARAMETER ON THE STACK (STACK)        

จะใช้ BP REGISTER (BASE POINTER)

        

SP REGISTER (STACK POINTER)

        

- NEAR = STACK จะเก็บแต่ OFFSET เนื่องจาก CODE SEGMENT และ DATA SEGMENT เก็บอยู่ที่เดียวกัน

        

- FAR = STACK จะเก็บทั้ง SEGMENT และ OFFSET เนื่องจาก CODE SEGMENT และ DATA SEGMENT เก็บแยกกัน

        
การผ่านค่าพารามิเตอร์ใน STACK อย่างไร ในภาษา ASSEMBLY  

1. กำหนด พารามิเตอร์ ว่าแต่ละตัวเป็นกี่ BYTE A=4, B=6, C=8

  

2. กำหนด PROCEDURE เป็นอย่างไร (NEAR,FAR)

  

3. เก็บ BP เริ่มต้นลงใน STACK ก่อน

  

4. MOV SP ไปใส่ใน BP

  

5. MOV ค่า LOCATION ของ พารามิเตอร์ A,B,C ไปไว้ที่ AX,BX,CX

LOCATION ของพารามิเตอร์ = ADDRESS ของ BP REGISTER + DISPLACEMENTDISPLACEMENT = ค่าคงที่ของพารามิเตอร์ที่เรากำหนดขึ้น

  

6. POP BP

  

7. RET

  

8. ENDP

  

RETURNING VALUES

การ RETURN ค่าโดยผ่านค่าได้ 3 ทาง  

1. STACK

  

2. REGISTER

  

3. MEMORY

  
เช่น เมื่อเห็นคำสั่ง RET = RETURN ADDRESS  

เวลา PUSH จะใส่ LOWBYTE ก่อน เช่น 0CA8: 0014 จะเก็บค่า 41 ก่อน 00 และ A8 แล้ว จึงเก็บ 0C และต้อง PUSH OFFSET ก่อน SEGMENT

  
สิ่งที่ใช้ STACK บ่อย คือ  

1. INTERRUPT VECTOR

  

2. CALL

  

3. RECURSIVE

คำสั่ง AND

AND INSTRUCTION คือ คำสั่งคำนวณทางตรรกศาสตร์ของ โอเปอร์แรนด์ 2 ตัว
รูปแบบ AND DEST, SOURCE

DEST = ปลายทาง

SOURCE = ต้นทาง
FLAG ที่ถูกกระทำ = OF=0, PF,SF,ZF
FLAG ที่ไม่ต้องกำหนด = AF
OPERATOR = AND
OPERATION = DEST, SOURCE
หลักการของ AND คือ T AND T ได้ T หรือ 1 AND 1 ได้ 1
วิธีใช้ ตามลักษณะของข้อมูล
1. REGISTER กับ REGISTER จะเร็วเพราะผ่านทาง CPU ไม่ผ่าน MEMORY เพราะ REGISTER อยู่ใน CPU
2. REGISTER กับ MEMORY ทำเมื่อมีขนาดไม่พอ
3. REGISTER กับ VALUE เช่น AND AL, FC
เมื่อใช้ DEBUG DUMP จาก NEMONIC CODE (INSTRUCTION) จะแปลงเป็น MACHINE CODE มีขั้นตอนดังนี้
ตรวจสอบโดยใช้ DEBUG

- A100

XXXX: 0100 AND AL,FC

- U100

XXXX 24 FC

- R ดูค่า REGITER เริ่มต้นก่อน

- T TRACE 1 ครั้ง

- R ดูค่า REGISTER สิ้นสุดจากการ AND

 

RECURSIVE

RECURSIVE คือการเรียกตัวเอง ทำงานซ้ำกัน เป็นหลักการเขียนโปรแกรมทำปัญหาใหญ่ให้เป็นปัญหาย่อยโดยมีลักษณะปัญหาเดียวกันเช่น N!

ข้อดี เขียนโปรแกรมง่าย

ข้อเสีย เปลือง STACK ไม่สามารถ CONTROLได้

INTERACTIVE คือ การเขียนธรรมดา CONTROL ได้เร็ว แต่เขียนโปรแกรมยาว

ภาษา โคบอล ฟอร์แทรน เขียนโปรแกรมแบบเรียกตัวเองไม่ได้

ภาษา ปาสคาล เบสิค ซี แอสเซมบลี้ เขียนโปรแกรมแบบเรียกตัวเองได้

การทำตัวอักษรเล็ก เป็นตัวอักษรใหญ่ ทำได้ 2 วิธี คือ

1. SUB AL, 20H

2. AND AL, 5FH
